Tally

A lightweight push-to-scrape metrics bridge for Prometheus.
POST a metric from any script or cron job — Prometheus scrapes it from /metrics.

/metrics /health README ↗

Endpoints

GET /metrics OpenMetrics scrape target for Prometheus
GET /health Liveness check — returns {"status":"ok"}
POST /push Push a single metric series — requires Authorization: Bearer <key>

Push payload

{
  "name": "cpu_usage_percent", // required — [a-zA-Z_][a-zA-Z0-9_]*
  "value": 72.4, // required — finite float
  "type": "gauge", // optional — gauge (default) | counter | histogram | summary | untyped
  "labels": { "instance": "web-1", "job": "cron" } // optional
}

One-liner examples

# Simple gauge — no labels
Invoke-RestMethod http://localhost:9200/push -Method POST \
  -Headers @{ Authorization = "Bearer YOUR_API_KEY" } \
  -ContentType "application/json" \
  -Body (ConvertTo-Json @{ name = "disk_used_bytes"; value = (Get-PSDrive C).Used })

# Counter with labels
Invoke-RestMethod http://localhost:9200/push -Method POST \
  -Headers @{ Authorization = "Bearer YOUR_API_KEY" } \
  -ContentType "application/json" \
  -Body (ConvertTo-Json @{ name = "jobs_total"; value = 42; type = "counter"; labels = @{ instance = "win-01" } })
# Simple gauge — no labels
curl -s -X POST http://localhost:9200/push \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \

# Counter with labels
curl -s -X POST http://localhost:9200/push \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"name":"jobs_total","value":42,"type":"counter","labels":{"instance":"web-1"}}'
